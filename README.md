# HealthDataProject
**Greek:**
**- Σχετικά με το πρόγραμμα: -**

Το πρόγραμμα HealthDataProject φτιάχτηκε στα πλαίσια της πτυχιακής εργασίας, που έχει ως θέμα την άντληση δεδομένων υγείας σε μορφή Open mHealth JSON σχήματα, τον μετασχηματισμό τους σε RDF γράφους, την αποθήκευση τους στην σημασιολογική βάση δεδομένων GraphDB και την προβολή τους μέσω μιας web εφαρμογής. Το συγκεκριμένο πρόγραμμα σε γλώσσα java υλοποιεί τον αυτόματο μετασχηματισμό των δεδομένων από JSON σχήματα (Open mHealth) σε τριπλέτες RDF, την αποθήκευση τους στο GraphDB και την επαλήθευσή τους με τη χρήση της SHACL.


**Περιεχόμενα του HealthDataProject:**
1. Στους φακέλους "Body Height", "Body Weight", "Calories Burned", "Heart Rate" "Physical Activity", "Speed", "Step Count" που βρίσκονται στον φάκελο "HealthDataProject\schema" τα αρχεία "body_height_temp.ttl", "body_weight_temp.ttl", "calories_burned_temp.ttl", "heart_rate_temp.ttl", "physical_activity_temp.ttl", "speed_temp.ttl", "step_count_temp.ttl" αντίστοιχα. Πρόκειται για template αρχεία με κανόνες RML και χρησιμοποιούνται από το πρόγραμμα για τη μετατροπή των JSON σχημάτων σε .ttl αρχεία με κανόνες RML.
2. Στον φάκελο "HealthDataProject\FinalOntology" θα αποθηκευτεί η τελική οντολογία που θα δημιουργηθεί μετά την εκτέλεση του προγράμματος
3. Στον φάκελο "HealthDataProject\FinalTTL" βρίσκονται αποθηκευμένα τα αρχεία sosa.ttl, qudt.ttl και time.ttl. Πρόκειται για τις οντολογίες SOSA, QUDT και TIME των οποίοων χρησιμοποιούνται οι κλάσεις, οι ιδιότητες και τα αντικείμενα, για το σχηματισμό της τελικής οντολογίας.
4. Στον φάκελο "HealthDataProject\GraphDBconfig" βρίσκεται το αρχείο config.ttl το οποίο χρησιμοποιείται για τη δημιουργία repository στο GraphDB.
5. Στον φάκελο "HealthDataProject\jars" βρίσκονται τα αρχεία json-20230227.jar και rmlmapper.jar, τα οποία πρέπει να εισαχθούν στον compiler για την εκτέλεση του προγράμματος.
6. Στον φάκελο "HealthDataProject\SHACLValidation" βρίσκεται το αρχείο shacl.ttl το οποίο χρησιμοποιείται για την επικύρωση της εγκυρότητας της οντολογίας.
7. Στον φάκελο "HealthDataProject\src\com\healthdata βρίσκονται οι κλάσεις του προγράμμτος.

**English:**
**- About the program: -**

The HealthDataProject program was built as part of the thesis, which is about retrieving health data in the form of Open mHealth JSON schemas, transforming them into RDF graphs, storing them in the semantic database GraphDB and viewing them through a web application. This java language program implements the automatic transformation of the data from JSON schemas (Open mHealth) to RDF triples, storing them in GraphDB and verifying them using SHACL.


**HealthDataProject:**
1. In the folders "Body Height", "Body Weight", "Calories Burned", "Heart Rate", "Physical Activity", "Speed", "Step Count" located in the folder "HealthDataProject\schema" the files "body_height_temp. ttl", "body_weight_temp.ttl", "calories_burned_temp.ttl", "heart_rate_temp.ttl", "physical_activity_temp.ttl", "speed_temp.ttl", "step_count_temp.ttl" respectively. These are template files with RML rules and are used by the program to convert JSON schemas into .ttl files with RML rules.
2. The folder "HealthDataProject\FinalOntology" will store the final ontology that will be created after running the program
3. In the folder "HealthDataProject\FinalTTL" the files sosa.ttl, qudt.ttl and time.ttl are stored. These are the SOSA, QUDT and TIME ontologies whose classes, properties and objects are used to form the final ontology.
4. In the folder "HealthDataProject\GraphDBconfig" is the file config.ttl which is used to create a repository in GraphDB.
5. In the folder "HealthDataProject\jars" are the files json-20230227.jar and rmlmapper.jar, which must be imported into the compiler to run the program.
6. In the folder "HealthDataProject\SHACLValidation" is the file shacl.ttl which is used to validate the validity of the ontology.
7. The folder "HealthDataProject\src\com\healthdata" contains the classes of the project.


**The classes included in the project are the following:**
1.	**ReadBodyHeight** class. This class reads the JSON shapes related to height and converts them first to RML format and then to RDF triples. The resulting triplets are stored in the FinalTTL folder in .ttl format with the name body_height_Measurement_Number_Name_Attribute_output.ttl (e.g. body_height_1_Nikos_output.ttl).
2.	**ReadBodyWeight** class. This class reads the JSON schemas related to weight and converts them first to RML format and then to RDF triples. The resulting triplets are stored in the FinalTTL folder in .ttl format with the name body_weight_measurement_Number_Name_Attribute_output.ttl (e.g. body_weight _1_Nikos_output.ttl).
3.	**ReadCaloriesBurned** class. This class reads the JSON schemas related to calories burned and converts them first to RML format and then to RDF triples. The resulting triplets are stored in the FinalTTL folder in .ttl format with the name calories_burned_Count_Number_Name_Output.ttl (e.g. calories_burned_1_Nikos_output.ttl).
4.	**ReadHeartRate class**. This class reads JSON schemas related to heartbeats and converts them first to RML format and then to RDF triples. The resulting triplets are stored in the FinalTTL folder in .ttl format with the name heart_rate_Count_Number_Count_Name_Attribute_output.ttl (e.g. heart_rate_1_Nikos_output.ttl).
5.	**ReadPhysicalActivity** class. This class reads the JSON schemas related to the activities and converts them first to RML format and then to RDF triples. The resulting triplets are stored in the FinalTTL folder in .ttl format with the name physical_activity_Count_Number_Measurement_NamePerson_output.ttl (e.g. physical_activity_1_Nikos_output.ttl).
6.	**ReadSpeed** class. This class reads the JSON schemas related to speed and converts them first to RML format and then to RDF triples. The resulting triplets are stored in the FinalTTL folder in .ttl format with the name speed_Count_Number_Name_Output.ttl (e.g. speed_1_Nikos_output.ttl).
7.	**ReadStepCount** class. This class reads the JSON schemas related to the steps taken and converts them first to RML format and then to RDF triples. The resulting triplets are stored in the FinalTTL folder in .ttl format with the name step_count_Count_Count_Name_Output.ttl (e.g. step_count_1_Nikos_output.ttl).
8.	**RMLMapper** class. This class converts files that are in RML format into RDF triples.
9.	**GraphDBManager** class. This class creates a repository in GraphDB using the config.ttl file in the GraphDBconfig folder, after checking if a repository with the same name already exists. It then uploads all files in the FinalTTL folder to the created repository.
10.	**ShaclValidation** class. This class is used to validate the RDF triples imported into the GraphDB repository. It uses the shacl.ttl file in the SHACLValidation folder, importing it into the repository that was created.
11.	**Class TTLFilesMerger**. This class is used to generate a single .ttl file that includes all .ttl files with RDF triples created. It essentially creates the final ontology, and stores it in the FinalOntology folder named HealthOntology.ttl. HealthOntology.ttl is used to view the ontology through the Protégé program.
12.	**Main** class. All other classes of the program are run from it in the order described.

**Instructions for use:**
1. Health data concerning the height of a person should be stored in the folder "HealthDataProject/schema/Body Height". A prerequisite is that these files are JSON schemas in Open mHealth format. Within the folder there are already 3 JSON schemas concerning the height of 3 different individuals. If the user wishes to run the program using their own measurements, they should delete the .JSON files inside the folder, and upload .JSON files relating to their own measurements. The saved files should have a name of the format body_height_Number_Name_User_Name.json (e.g. body_height_1_Dimitris.json).

A prerequisite is that these files are JSON schemas in Open mHealth format. Within the folder there are already 22 JSON schemas concerning the weight of 3 different individuals. 8. When running the program, the GraphDB Desktop program, which is the server that handles all interactions with the database, should be running in parallel.
9. To run the program, the json-20230227.jar and rmlmapper.jar stored in the "HealthDataProject\jars" folder need to be imported.
10. The name of the GraphDB repository to be created is defined within the config.ttl file located in the "HealthDataProject\GraphDBconfig" folder, with the command rep:repositoryID "Name". The name of the repository as configured in the config.ttl file is "HealthData". If the user wishes to use a different name, the name should be changed in the rep:repositoryID "HealthData" command in config.ttl
11. By running the program, in addition to creating the repository in GraphDB and storing the RDF triplets in it, a .ttl file is created, which is the final ontology, i.e. the union of all the RDF triplets. This file is stored in the folder "HealthDataProject\FinalOntology".


**The result of the execution of the program is as follows:**
1. In the folders "Body Height", "Body Weight", "Calories Burned", "Heart Rate", "Physical Activity", "Speed", "Step Count" located in the folder "HealthDataProject\schema" for each .JSON file that exists, a .ttl file with RML rules is created
2. In the folder "HealthDataProject\FinalTTL" all .ttl files containing RDF triples resulting from the transformation of JSON files with RML rules are stored.
3. The folder "HealthDataProject\FinalOntology" stores the final ontology, with the union of all .ttl files present in the folder "HealthDataProject\FinalTTL".
4. A repository is created in GraphDB into which all files in the "HealthDataProject\FinalTTL" folder are uploaded.
5. In the GraphDB repository created, the shacl.ttl file of the "HealthDataProject\SHACLValidation" folder is uploaded to perform the validation of the RDF triplets.
